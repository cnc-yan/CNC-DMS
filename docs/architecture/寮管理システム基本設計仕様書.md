# **1\. プロジェクト概要**

## **1.1 プロジェクト背景**

現在、社内では Excel と手作業による従業員寮の管理を行っており、以下の課題が存在します。

* 空き状況（部屋のステータス）情報の非同期  
* 入寮申請・承認の非効率さ  
* データの重複およびエラーの頻発  
* 空き室管理の不透明さ  
* 寮費計算の複雑さ  
* 監査トレース（追跡）の困難さ

これらの課題を解決するため、統一された企業向けの寮管理システムを構築し、以下を実現します。

* 入退寮プロセスの電子化  
* 部屋のステータスのリアルタイム同期  
* 承認プロセスの標準化  
* Excel による一括インポート  
* 監査ログの追跡可能性  
* 寮費の自動計算

# **10\. Excel インポートおよびバッチ設計**

## **10.1 Excel インポートプロセス**

Plaintext  
アップロード (Upload)  
 ↓  
フォーマットバリデーション（形式チェック）  
 ↓  
ビジネスロジックバリデーション（整合性チェック）  
 ↓  
重複チェック  
 ↓  
一括書き込み（バルクインサート）  
 ↓  
インポート結果レポートの生成

## **10.2 インポートバリデーションルール**

* 部屋番号は必須（空不可）  
* 従業員 ID がシステム内に存在すること  
* 入寮日が有効な日付であること  
* 性別が寮の制限タイプと一致していること  
* 重複する入寮（二重入居）は不可

## **10.3 バッチ（Batch）ジョブ**

| billing-batch | 寮費の自動生成 |
| vacancy-batch | 空き室情報の自動更新 |
| report-batch | 統計レポートの自動生成 |

# **14\. 異常系・耐障害性設計**

## **14.1 グローバル例外処理**

* 生の SQL エラーは画面に露出させないこと  
* 統一された業務エラーコードを返却すること

## **14.2 エラーコード一覧**

| エラーコード | 説明（理由） |
| :---- | :---- |
| E4001 | ROOM\_FULL（満室） |
| E4002 | DUPLICATE\_CHECKIN（重複入寮） |
| E4003 | INVALID\_BILLING（不正な請求データ） |
| E4004 | ROOM\_GENDER\_CONFLICT（部屋の性別不一致） |

# **15\. ログおよび監査設計**

## **15.1 監査項目（フィールド）**

| 項目名 | 説明 |
| :---- | :---- |
| operator | 操作者 |
| operation\_type | 操作タイプ |
| before\_value | 変更前値 |
| after\_value | 変更後値 |
| request\_ip | IP アドレス |
| trace\_id | トレース ID / コンテキスト ID |

## **15.2 監査要件**

* すべてのデータ変更（更新・削除）は必ず記録すること  
* データ削除を行う場合は論理削除を採用すること  
* インポート処理は結果ログを完全に保持すること
